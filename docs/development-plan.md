# 媒体管理器（WinUI 3）开发计划

> 目标：基于 WinUI 3 + Windows App SDK 重建媒体管理器，优先实现核心播放与管理能力，再扩展高级功能。

## 里程碑 0：工程与架构（已完成）
1. 建立 WinUI 3 工程结构与基础 MVVM 架构。
2. SQLite 数据层初始化与基础数据模型。
3. 基础媒体扫描导入与播放显示。
4. 搜索、排序、列表/网格视图切换、基础快捷键。

## 里程碑 1：媒体库核心完善
1. 扫描进度显示与状态提示（替换当前简单文字）。
2. 拖拽导入（文件/文件夹）。
3. 缩略图缓存策略与清理入口（磁盘缓存）。
4. 右键菜单（媒体项：打开目录、删除、编辑标签、添加到播放列表）。
5. 媒体批量选择与批量操作。

## 里程碑 2：标签系统
1. 标签编辑器（单个/多个媒体）。
2. 标签搜索（#标签 + 多标签 AND 过滤）。
3. 标签列表管理（可选：全局标签维护）。

## 里程碑 3：播放列表
1. 播放列表 CRUD（创建/重命名/删除/排序）。
2. 添加/移除媒体到播放列表。
3. 播放列表内搜索与排序。
4. 可选：播放列表独立窗口。

## 里程碑 4：播放体验升级
1. 自定义播放器控制条（进度、时间、音量、全屏）。
2. 播放模式（列表循环/单曲循环/随机）。
3. 播放结束自动切换逻辑。
4. 图片查看器：缩放、拖拽、快捷键。

## 里程碑 5：视频剪辑
1. 剪辑界面：多段区间编辑（拖拽 + 输入）。
2. 剪辑模式：保留/删除。
3. 输出目录选择与进度反馈。
4. 剪辑失败恢复与提示。

## 里程碑 6：设置与系统功能
1. 数据目录切换。
2. 便携模式支持。
3. 缓存管理（缩略图/临时文件/全量清理）。
4. 文件夹锁定与密码管理。

## 里程碑 7：性能与稳定性
1. 列表/网格虚拟化优化。
2. 大库性能测试（万级媒体）。
3. 异常恢复与错误日志。
4. UI 细节打磨（动画、主题、状态反馈）。

## 验收标准（每个里程碑均需满足）
1. 功能完整可用，且符合需求文档描述。
2. 无明显卡顿或阻塞，UI 响应及时。
3. 关键操作具备明确提示与错误处理。
4. 基础快捷键全部可用。

## 风险与注意事项
1. WinUI 3 中部分功能（如拖拽、缩略图）需要额外权限或 API 适配。
2. FFmpeg 集成方案需确认（WinUI 3 可考虑外部进程或本地库封装）。
3. 播放列表独立窗口与主窗口数据同步需要清晰的数据流设计。
