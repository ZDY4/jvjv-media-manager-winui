# 暗色 UI 优化拆解清单

## 目标

将当前界面逐步调整到更接近 Windows 11 照片查看器和媒体播放器的暗色风格。

设计基线：

- 统一暗色画布
- 弱边界
- 媒体优先
- 轻工具栏
- 圆角浮层
- 仅暗色主题

## 执行方式

- 每次只选择 1 个点实施。
- 每次实施后都要运行应用并查看实际效果。
- 你确认“这个点通过”后，再进入下一个点。
- 如果某个点实施后发现牵连过大，就把它继续拆小，再按更小粒度推进。

默认优先顺序：

`2 -> 3 -> 4 -> 6 -> 7 -> 9 -> 15 -> 17 -> 21 -> 29 -> 34`

## 约束

- 整个项目只做暗色风格，不做亮色模式切换和亮色校验。
- 讨论和实施都以 WinUI 3 原生 Fluent 暗色风格为基线，不引入第三方 UI 框架。
- 每次改动都尽量只碰一个视觉主题，避免多个点叠加后难以判断效果来源。

## 清单

- [ ] 1. 建立新的视觉基线说明  
  明确目标气质为“统一暗色画布、弱边界、媒体优先、轻工具栏、圆角浮层、仅暗色主题”，并把这段说明作为后续所有改动的判断标准。

- [x] 2. 收敛全局暗色色板  
  将全局颜色从当前多组近似深灰整理为更少的层级色，保留单一强调色，明确背景层、表面层、浮层、描边、主文字、次文字的用途。

- [ ] 3. 改造全局圆角体系  
  统一应用中的小、中、大圆角规则，移除当前大量 `CornerRadius="0"` 的硬边视觉，优先让媒体项、浮层、输入框、控制条变得更接近 Windows 11。

- [ ] 4. 改造全局按钮风格  
  将当前较重的自定义按钮模板收敛为更轻的 WinUI 风格，明确主按钮、次按钮、图标按钮的层级，减少“每个按钮都像独立实体块”的感觉。

- [ ] 5. 改造输入框和筛选控件风格  
  让搜索框、密码框、下拉框从硬编码深色矩形改成更系统、更圆润、更轻的输入面板。

- [ ] 6. 调整窗口背景和材质感  
  重新定义主窗口背景、环境光渐变和表面材质，目标是接近 Windows 11 暗色应用壳层，而不是单纯一整片深灰。

- [ ] 7. 重构标题栏布局  
  让标题栏更轻，减少顶部工具按钮的压迫感，强化标题区和命令区的层次。

- [ ] 8. 重新组织标题栏命令优先级  
  保留“添加文件 / 添加文件夹 / 设置”能力，但调整它们的视觉权重和排列方式，避免与窗口标题争夺焦点。

- [ ] 9. 去掉主界面的“双大卡片”观感  
  让左侧库区和右侧播放器区更像统一画布里的分区，而不是两个并排的大盒子。

- [ ] 10. 调整主页面左右分区间距  
  重新定义 `MainPage` 外边距、左右区间距、内边距，让界面更通透，减少目前略紧的工具面板感。

- [ ] 11. 优化左侧播放列表 rail 的整体气质  
  让 rail 更接近系统导航条，弱化单个色块按钮的突兀感，提升选中态与未选中态的对比方式。

- [ ] 12. 优化播放列表 rail 的选中态  
  将当前偏色块覆盖的选中反馈改成更像 WinUI 的柔和高亮或指示器风格。

- [ ] 13. 优化库区头部层级  
  重新安排播放列表标题、刷新、视图切换、排序和扫描状态的视觉主次，避免头部信息拥挤。

- [ ] 14. 优化扫描状态显示  
  让扫描进度、状态文本、路径提示更轻、更易扫读，不与主操作区争抢注意力。

- [ ] 15. 重构搜索栏视觉  
  提升搜索框的可识别性与精致感，让它更像系统媒体应用的搜索入口，而不是普通工具输入框。

- [ ] 16. 优化标签筛选 chip  
  统一标签 chip 的圆角、颜色、选中与可点击反馈，减少当前略显普通的按钮感。

- [ ] 17. 降低媒体列表项的卡片感  
  列表项改为“内容优先”的行项设计，减少实体底板存在感，强化缩略图、标题、元信息的层级。

- [ ] 18. 优化媒体列表项选中态  
  让选中反馈更清晰但更克制，避免整块强调色遮罩影响内容观感。

- [ ] 19. 调整媒体列表项的缩略图比例和留白  
  让缩略图更像照片应用中的内容预览，而不是文件列表里的附属小图。

- [ ] 20. 优化媒体列表项文本排版  
  重新定义文件名、尺寸、分辨率、时长、标签的字号和字重关系，提高扫读效率。

- [ ] 21. 降低媒体网格项的卡片感  
  让网格项更偏照片流而不是卡片墙，减少外层实体框与硬边结构。

- [ ] 22. 优化媒体网格项底部信息层  
  让底部信息遮罩更轻、更精致，减少厚重黑条感。

- [ ] 23. 优化媒体网格项选中态  
  使用更贴近系统媒体应用的描边、柔和遮罩或角标方式表达选中。

- [ ] 24. 优化媒体网格的密度和间距  
  调整 item 宽高、间距和边缘留白，让浏览体验更像媒体浏览器而不是文件缩略图面板。

- [ ] 25. 重构播放器主画布背景  
  让视频和图片承载区更干净，让媒体内容真正成为视觉中心。

- [ ] 26. 优化播放器信息浮层  
  让左上角文件名和分辨率浮层更轻、更圆、更系统，不像独立深色矩形块。

- [ ] 27. 优化图片缩放角标  
  让缩放提示更弱打扰但可读，和其他浮层使用统一材质与圆角语言。

- [ ] 28. 优化左右翻页提示  
  让图片切换 cue 更像 Windows 11 照片查看器的边缘提示，而不是小型深色柱块。

- [ ] 29. 重构底部播放控制条外观  
  让控制条更接近 Windows 11 媒体播放器的浮动控制面板，统一圆角、透明度、边框和内边距。

- [ ] 30. 重新平衡播放控制条里的控件权重  
  强化播放按钮和进度条，弱化次要按钮，减少每个图标按钮都同样抢眼的问题。

- [ ] 31. 优化进度条和音量条样式  
  让 `Slider` 更贴近系统媒体类控件风格，而不是默认工具条滑块。

- [ ] 32. 优化音量弹出层样式  
  统一音量弹层与控制条的材质和圆角，避免当前完全方正的弹出层。

- [ ] 33. 优化剪辑条视觉  
  在不改变功能入口的前提下，让剪辑栏与播放器控制条同属一个视觉系统，而不是独立的另一套面板。

- [ ] 34. 重做空态界面  
  把空态从居中卡片改成更像画布提示，让图标、标题和说明更接近照片应用的轻提示气质。

- [ ] 35. 统一设置、锁定、标签、剪辑方案等面板风格  
  让二级面板和对话内容继承同一套圆角、表面和控件体系，避免主界面变好了、弹层还停留在旧风格。

- [ ] 36. 检查键盘焦点和 hover 反馈  
  确保重构后所有重要控件仍有清晰的焦点、hover、pressed 状态，符合 WinUI 使用习惯。

- [ ] 37. 检查窗口宽度变化下的布局稳定性  
  验证在窄窗口、中窗口、宽窗口下，标题栏、库区、控制条和媒体网格的密度与布局不会失衡。

- [ ] 38. 增加轻量过渡动画  
  只补充少量必要动效，例如媒体切换淡入、浮层显隐、视图切换过渡，避免界面静态而生硬。

- [ ] 39. 做最终统一性巡检  
  最后一轮检查颜色、圆角、阴影、间距、字号、状态反馈是否已统一成单一设计语言。
