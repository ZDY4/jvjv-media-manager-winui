# UI Modernization Backlog

更新日期：2026-03-22

本文记录当前尚未完成、但已经明确要继续推进的界面优化项。

## High Priority

- 为媒体 `ListView` / `GridView` 增加更细腻的 hover / pressed / focus 动画。
- 为缩略图加载增加骨架屏或占位过渡，减少大批量媒体首次加载时的突兀感。
- 为浅色主题逐页做人工校色，确认播放器、标签 chip、悬浮控制条和选中态在亮背景下依然清晰。
- 把播放器区域的顶部信息浮层做成更完整的信息卡，包括文件名、分辨率、时长、标签等摘要。
- 统一上下文菜单、对话框标题区和按钮布局节奏，减少不同弹窗之间的视觉差异。

## Medium Priority

- 为媒体卡片增加更明显但克制的选中、当前播放、锁定状态标识。
- 优化 `SplitView` 左侧面板在窄窗口下的响应式布局，避免按钮和列表过于拥挤。
- 重做播放列表项样式，增加更强的层次感、选中态和拖拽排序反馈。
- 为空状态增加更统一的插图语言或品牌图形资源，而不是完全依赖内置图标。
- 为设置、标签编辑、片段方案、锁定管理等弹窗补统一的说明文字和状态提示样式。

## Polish

- 梳理全局图标系统，统一线条粗细、语义映射和尺寸。
- 为标题栏、顶部 hero 区和播放器控制条增加更细微的动态反馈。
- 为浅色 / 深色模式分别检查阴影、描边和透明度层级，避免出现“发灰”或“发脏”。
- 补充界面截图和设计说明到项目文档，方便后续继续迭代。
- 评估是否需要引入更完整的设计 token 命名规范，进一步收敛颜色和间距资源。

## Verification Checklist

- 每次继续 UI 改造后都重新执行 `pwsh -NoLogo -NoProfile -File .\build.ps1`。
- 在深色、浅色、跟随系统三种模式下分别检查主页面、播放器、设置面板和弹窗。
- 至少验证 1366x768 和 1600x900 两档窗口尺寸下的主要布局表现。
